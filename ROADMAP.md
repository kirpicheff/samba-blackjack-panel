# Дорожная карта развития Samba Blackjack Panel

Этот документ описывает план развития панели от текущего состояния до полнофункционального инструмента администрирования.

## ✅ Этап 0: Безопасность и авторизация (Завершено)
*Цель: Ограничить доступ к панели управления.*

- [x] **Страница входа:** Создание UI для ввода логина и пароля.
- [x] **Middleware:** Защита всех API-маршрутов проверкой сессии.
- [x] **Управление сессиями:** Выход из системы (Logout) и хранение состояния.

## ✅ Этап 1: Управление ресурсами (Full CRUD) (Завершено)
*Цель: Дать возможность полностью настраивать общие папки без ручного редактирования smb.conf.*

- [x] **Создание ресурса:** Модальное окно с выбором пути, имени и базовых прав.
- [x] **Редактирование параметров:** Настройка `read only`, `guest ok`, `browseable`, `valid users`, масок прав и ACL.
- [x] **Управление VFS:** Удобное включение/выключение модулей (детальная настройка `recycle` для корзины).
- [x] **Удаление ресурса:** Безопасное удаление секции из конфига с подтверждением.

## ✅ Этап 2: Управление пользователями и группами (Завершено)
*Цель: Управление доступом пользователей.*

- [x] **Просмотр пользователей:** Список аккаунтов через `pdbedit`.
- [x] **Создание/удаление пользователей:** Интерфейс для `smbpasswd` и `pdbedit`.
- [x] **Смена паролей:** Возможность быстро обновить пароль пользователя.
- [x] **Управление группами:** Создание групп Samba и привязка пользователей к ним.

## ✅ Этап 3: Глобальные настройки и контроль сервиса (Завершено)
*Цель: Управление поведением всего сервера.*

- [x] **Секция [global]:** Редактор имени сервера, рабочей группы, протоколов, сетевых интерфейсов и безопасности.
- [x] **Управление состоянием:** Кнопки «Старт», «Стоп», «Рестарт» и «Применить изменения».
- [x] **Мониторинг сервиса:** Отображение реального статуса smbd (Running/Stopped) на Дашборде.
- [x] **Валидация:** Автоматический запуск `testparm` перед сохранением изменений.

## ✅ Этап 4: Расширенный мониторинг и аудит (Завершено)
*Цель: Понимание того, что происходит на сервере в реальном времени.*

- [x] **Графики нагрузки:** Мониторинг использования дискового пространства для всех ресурсов.
- [x] **Лог-вьюер:** Просмотр логов Samba в реальном времени прямо в панели.
- [x] **Аудит (VFS full_audit):** Отображение истории действий: кто, когда и какой файл открыл/удалил.

## 🚀 Этап 6: Сетевая интеграция и диагностика (В ПЛАНЕ)
- [x] **Сетевое обнаружение (WSDD/Avahi):** Управление сервисами для автоматического появления сервера в окружении Windows (Network) и macOS (Finder).
- [x] **Диагностика AD (Health Check):** Инструментарий для проверки связи с контроллером домена, синхронизации времени и проверки Kerberos-билетов.

## 🛡️ Этап 7: Безопасность и UX мониторинга
- [x] **Ограничение по IP:** Визуальный редактор `hosts allow` / `hosts deny` для ограничения доступа к ресурсам на уровне подсетей.
- [x] **Live Logs (WebSockets):** Перевод просмотра логов на Real-time поток, чтобы видеть активность без обновления страницы.
- [x] **Управление правами ФС (ACL):** Возможность менять владельца (chown) и маску прав (chmod/facl) для папок ресурсов прямо из UI.
- [x] **Многоязычность (i18n):** Поддержка переключения интерфейса на английский (EN) и русский (RU) языки с сохранением выбора.

## 📅 Этап 8: Будущие расширения (Backlog)
- [ ] **Встроенный файловый менеджер:** Базовые операции с файлами на сервере (создание папок, переименование). Совмещение с визуальным управлением правами (ACL).
- [ ] **Дисковые квоты:** Ограничение использования места конкретными пользователями или отделами.
