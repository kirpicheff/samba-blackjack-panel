## 🚀 Быстрая установка (Ubuntu/Debian)

Самый простой способ установить панель как системную службу:

```bash
wget https://raw.githubusercontent.com/kirpicheff/samba-blackjack-panel/main/install.sh
chmod +x install.sh
sudo ./install.sh
```

После установки панель будет доступна по адресу: `http://ваш-ip:8888`.
- **Логин:** `admin`
- **Пароль:** `admin` (смените сразу после входа в "Настройки сервера" -> "Администраторы панели")

## ✨ Возможности

### 📊 Дашборд и Мониторинг
- **Активные сессии**: Просмотр подключенных пользователей, их IP и версий протокола SMB.
- **Открытые файлы**: Список всех файлов, используемых клиентами в данный момент.
- **Статус дисков**: Мониторинг свободного места на разделах с общими папками.
- **Службы**: Контроль состояния WSDD (Windows) и Avahi (macOS/Linux).

### 📂 Управление ресурсами (Shares)
- **Создание и редактирование**: Полное управление секциями `smb.conf` через UI.
- **Корзина (Recycle Bin)**: Автоматическая очистка, исключение файлов, настройка путей.
- **Аудит**: Логирование действий (удаление, переименование) во встроенный журнал.
- **Права ФС (ACL)**: Визуальный редактор владельца, группы и прав доступа (`chmod`) с восьмеричной маской и матрицей чекбоксов.
- **Ограничение по IP**: Настройка `hosts allow` и `hosts deny` как глобально, так и для конкретных ресурсов.
- **Теневые копии**: Поддержка VFS Shadow Copy 2 для восстановления файлов.
- **Дисковые квоты**: Управление лимитами (Soft/Hard) для пользователей и групп с визуальным мониторингом использования.

### 👥 Пользователи и Группы
- **Samba Users**: Управление аккаунтами через `pdbedit` (создание, пароли, удаление).
- **Группы ОС**: Управление системными группами и членством пользователей.

### 🌐 Active Directory (AD)
- **Авто-ввод в домен**: Настройка Kerberos, Winbind и выполнение Join.
- **Health Check**: Глубокая диагностика связи с контроллером домена (Trust, Time, RPC, Keytab).

### ⚙️ Настройки и Безопасность
- **Глобальные параметры**: Настройка Workgroup, Netbios Name, сетевых интерфейсов.
- **Контроль сервисов**: Start/Stop/Restart для smbd, nmbd, winbind.
- **Авто-бэкапы**: Хранение последних 10 версий `smb.conf`.
- **Доступ к панели**: Многопользовательский вход, Bcrypt-хеширование паролей, управление админами.

### 📜 Логи и Автоматизация
- **Live Logs**: Просмотр системных логов `log.smbd` в реальном времени через WebSockets.
- **Файловый менеджер**: Полноценный встроенный менеджер для создания папок, переименования и удаления файлов прямо из браузера.
- **Журнал аудита**: Просмотр истории действий пользователей во встроенной таблице.
- **Фоновые задачи**: Автоматическая очистка корзин и создание снимков (snapshots).

## 💻 Режим разработки (Windows/macOS)

Если вы запускаете панель не на Linux, она автоматически переходит в **Mock-режим**:
- Используются тестовые данные вместо реальных системных вызовов.
- Создается локальный файл `smb.conf.dev` для имитации конфигурации.

**Запуск для разработки:**
1. Убедитесь, что установлен Go.
2. Склонируйте репозиторий.
3. Выполните: `go run .`
4. Откройте `http://localhost:8888`.

## 🛠 Ручная установка и восстановление

### Системные требования (Linux)
Для работы всех функций необходимо:

> [!IMPORTANT]
> **Дисковые квоты:** Для работы квот на Linux, раздел должен быть смонтирован с опциями `usrquota` и `grpquota` в `/etc/fstab`. 
> Пример: `/dev/sdb1 /shares ext4 defaults,usrquota,grpquota 0 2`

```bash
sudo apt update
sudo apt install samba samba-common-bin krb5-user winbind avahi-daemon acl quota
```

### Восстановление доступа
Пароли администраторов панели хранятся в `admins.json`. Если вы потеряли доступ:
1. Удалите файл `admins.json`.
2. Перезапустите панель.
3. Используйте стандартные данные: `admin / admin`.
