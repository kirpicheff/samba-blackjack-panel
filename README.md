# Samba Blackjack Panel

Легковесная панель управления сервером Samba на Go.

## Возможности

### 📊 Дашборд и Мониторинг
- **Активные сессии**: Просмотр подключенных пользователей, их IP и версий протокола SMB.
- **Открытые файлы**: Список всех файлов, используемых клиентами в данный момент.
- **Статус дисков**: Мониторинг свободного места на разделах с общими папками.
- **Службы**: Контроль состояния WSDD (Windows) и Avahi (macOS/Linux).

### 📂 Управление ресурсами (Shares)
- **Создание и редактирование**: Полное управление секциями `smb.conf` через UI.
- **Корзина (Recycle Bin)**: Автоматическая очистка, исключение файлов, настройка путей.
- **Аудит**: Логирование действий (удаление, переименование) во встроенный журнал.
- **Теневые копии**: Поддержка VFS Shadow Copy 2 для восстановления файлов.

### 👥 Пользователи и Группы
- **Samba Users**: Управление аккаунтами через `pdbedit` (создание, пароли, удаление).
- **Группы ОС**: Управление системными группами и членством пользователей.

### 🌐 Active Directory (AD)
- **Авто-ввод в домен**: Настройка Kerberos, Winbind и выполнение Join.
- **Health Check**: Глубокая диагностика связи с контроллером домена (Trust, Time, RPC, Keytab).

### ⚙️ Настройки и Безопасность
- **Глобальные параметры**: Настройка Workgroup, Netbios Name, сетевых интерфейсов.
- **Контроль сервисов**: Start/Stop/Restart для smbd, nmbd, winbind.
- **Авто-бэкапы**: Хранение последних 10 версий `smb.conf`.
- **Доступ к панели**: Многопользовательский вход, Bcrypt-хеширование паролей, управление админами.

### 📜 Логи и Автоматизация
- **Просмотр логов**: Прямое чтение `log.smbd` и журнала аудита из интерфейса.
- **Фоновые задачи**: Автоматическая очистка корзин и создание снимков (snapshots).

## Запуск
1. Убедитесь, что у вас установлен Go.
2. Склонируйте репозиторий.
3. Запустите проект:
   ```bash
   go run .
   ```
4. Откройте в браузере `http://localhost:8888`.

## Требования и зависимости (Linux)

Для полноценной работы всех функций панели в Linux-системе (рекомендуется Ubuntu/Debian) необходимо установить следующие пакеты:

### 1. Основные (База)
```bash
sudo apt update
sudo apt install samba samba-common-bin
```
*   `smbstatus` — для мониторинга сессий.
*   `pdbedit` — для управления пользователями.

### 2. Интеграция с Active Directory (Этап 5)
```bash
sudo apt install krb5-user winbind libnss-winbind libpam-winbind
```
*   Необходимо для работы команды `net ads join` и авторизации через домен.

### 3. Сетевое обнаружение (Этап 6)
```bash
sudo apt install avahi-daemon
# wsdd обычно ставится из сторонних репозиториев или скачивается скриптом
sudo apt install wsdd 
```
*   `avahi-daemon` — для обнаружения в macOS/Linux (mDNS).
*   `wsdd` — для обнаружения в Windows 10/11 (Web Services Dynamic Discovery).

### 4. Дополнительные модули
*   `samba-vfs-modules` — требуется для работы Аудита (`full_audit`) и Корзины (`recycle`).
*   `acl` — для управления расширенными правами доступа (`getfacl`/`setfacl`).

## Управление администраторами

При первом запуске панель создает файл `admins.json` с учетными данными по умолчанию:
- **Логин:** `admin`
- **Пароль:** `admin`

**Рекомендуется сменить пароль сразу после первого входа** в разделе "Настройки сервера" -> "Администраторы панели".

Пароли хранятся в виде bcrypt-хешей. Если вы забыли пароль и потеряли доступ, просто удалите файл `admins.json` и перезапустите панель — она снова создаст стандартного пользователя `admin/admin`.

## Режим разработки (Windows/macOS)
Если вы запускаете панель не на Linux, она автоматически переходит в **Mock-режим**:
- Вместо реальных системных вызовов используются заглушки с тестовыми данными.
- Создается локальный файл `smb.conf.dev` для имитации конфигурации.
